Transportable Detonation Chamber

v1.0 — Malware Analysis & EDR Testing Platform

What is Detonation Chamber?

A pre-configured Windows 11 VM for malware detonation testing against multiple EDR solutions. The unified web UI aggregates telemetry from all engines into a single dark-themed interface for real-time analysis.

Key Capabilities

🔍
Real-time Detection
Sigma rules, YARA signatures, and IOC hash matching via Rustinel ETW engine
🧬
Deep Telemetry
Kernel-level ETW tracing via Fibratus — process, file, registry, network events
🛡
Multi-Engine Scanning
PE-Sieve, Moneta, HollowsHunter, RedEdr, ThreatCheck, DefenderCheck
📊
PE Analysis
Header parsing, entropy detection, import/export analysis, packer identification

Workflow

  1. Submit a sample via the Submit tab (drag & drop or browse)
  2. DetonatorAgent executes the sample in the VM
  3. LitterBox runs static analysis (YARA, strings) + dynamic analysis (PE-Sieve, Moneta, HollowsHunter)
  4. Fibratus + Rustinel capture ETW telemetry and fire Sigma/YARA detection rules
  5. View results in real-time across Dashboard, Tracing, Graph, and Sysmon tabs

Tab Guide

Dashboard

System overview with stats strip (alerts, processes, services, rules, tools), service health cards, and recent activity feed. Auto-refreshes every 10 seconds.

Tracing

Real-time console showing all ETW events and detection alerts. Select a process from the sidebar to filter events. Timeline visualization shows alert distribution over time.

Graph

Process relationship visualization with 5 layout modes (Force-directed, Hierarchical, Radial, Circular, Grid). Zoom controls, search, and time-range filtering. Nodes represent processes, edges show parent-child relationships.

Sysmon

Windows Sysmon event log viewer. Shows process creation, network connections, file creation, and registry events from Sysmon's ETW provider.

Scanner

Run ThreatCheck or DefenderCheck against files. Supports Defender and AMSI engines. Results show detection status, byte offset of trigger, and link to hex editor for inspection.

Hex Editor

Binary file viewer with offset navigation, data inspector (Int8/16/32, Float, ASCII, UTF-16), and drag-and-drop file upload. PE Analysis button parses headers, sections, imports, and flags IOC indicators.

Submit

Detonate samples against DetonatorAgent and/or LitterBox. Triggers full analysis pipeline and polls for results. Shows stage-by-stage progress with Fibratus alerts, static analysis (YARA), and dynamic analysis (PE-Sieve, Moneta, HollowsHunter).

Data Flow

  1. Sample submitted via Web UI → forwarded to DetonatorAgent + LitterBox
  2. DetonatorAgent executes the sample, returns PID
  3. LitterBox runs static (YARA, CheckPlz, Stringnalyzer) + dynamic (PE-Sieve, Moneta, HollowsHunter, RedEdr) analysis
  4. Fibratus captures kernel-level ETW events for the process
  5. Rustinel matches events against 20 Sigma + 717 YARA rules + IOC hashes
  6. Web UI aggregates alerts from all engines into unified timeline

Detection Rules

Sigma Rules (20)
Process injection, credential dumping, lateral movement, persistence, defense evasion patterns
YARA Rules (717)
Compiled rule set covering malware families, packers, exploits, shellcode patterns
IOC Engine
Hash-based matching. Auto-fed when samples are submitted. Supports SHA-256.

PE Analysis Capabilities

  • Headers: DOS, File, Optional (PE32/PE32+), section table
  • Security: ASLR, DEP/NX, SEH, CFG status
  • Entropy: Per-section Shannon entropy with color-coded bars (>7.0 = packed/encrypted)
  • Imports: Categorized suspicious API detection (injection, evasion, credential access, networking)
  • Packers: Section name matching (UPX, Themida, VMProtect, ASPack, MPRESS, etc.)
  • TLS Callbacks: Anti-debug indicator detection
  • RWX Sections: Read+Write+Execute flagging
